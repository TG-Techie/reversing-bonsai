## Within-block ordering hypothesis: attn-before-MLP (synthesis of 40-43)

This document gathers the evidence from `40_*` through `43_*` plus
the cross-size data from `local-1.7B/42_*` to argue for one specific
recipe constraint: **the technique processes attention before MLP
within each transformer block**.

The argument is the simplest explanation for the asymmetric MLP-vs-
attention pattern at L1-3.

## Restated observations (force-by-data)

1. **L1-3 MLP gate/up over-dispersion at 8B is 10-13** vs baseline
   ~1.1 elsewhere (`40_*`).
2. **L1-3 attention at 8B has NO over-dispersion spike** —
   q stays at 1.78-2.10 (in-range with q's 1.7-3.2 across all
   depths) (`40_*`).
3. **L1-3 MLP rank-concentration is elevated** — rank-128 = 10-12%
   at L1-L2 vs baseline 7.5-8% (`41_*`).
4. **L1-3 attention rank-concentration is NOT elevated** — q
   rank-128 = 13.27-14.68% across L0/L1/L2/L3/L18/L35 (1.4pp
   spread) (`43_*`).
5. **At 1.7B, the same MLP-only L1 elevation pattern appears
   modestly** — `up` jumps to 2.47 at L1, `gate` to 1.57, vs L0
   1.09 / 1.26. Attention barely changes (q L0=2.29, L1=2.31).
   Magnitude much smaller than 8B but the asymmetric MLP-vs-attn
   pattern is preserved (`local-1.7B/42_*`).
6. **Approximately full-rank delta everywhere** — `34_*`, `39_*`,
   `41_*`, `43_*`. No projection at any depth has rank-128 % > 28%.
7. **Per-block flips ARE block-correlated** — over-dispersion
   1.07-13× across (depth, type) (`40_*`).

## Why "attn-before-MLP within each block" is the simplest fit

In a transformer block at layer L:
```
residual_in -> attn(residual_in) -> add -> mlp(attn_out + residual_in) -> add -> residual_out
```

A sequential per-tensor quantisation pass that processes:
```
... attn_at_L1, mlp_at_L1, attn_at_L2, mlp_at_L2, ...
```

means MLP at L1 makes its per-block decisions with an input
distribution that includes:
- L0-block's full quantisation error (compounded into the residual
  stream)
- **L1-attention's quantisation error** (added at L1's intra-block
  attn output)

Whereas attn at L1 makes its decisions with input that includes
only L0-block's error (no within-block extra).

If per-block decisions in each tensor are made to minimise
activation-output reconstruction error against the (quantised) input
they actually see, then:
- MLP at every layer sees more error than attention at the same
  layer
- The relative magnitude of this within-block extra-error vs total
  accumulated error is largest at early layers (where total
  accumulated error is small) and shrinks at deep layers (where
  total error dominates)

Predictions:
- (a) MLP at every layer should be more block-coherent than
  attention at the same layer.
- (b) The MLP-vs-attn coherence GAP should be biggest at early
  layers.
- (c) Deep-layer MLP and attention coherence should converge
  (within-block extra-error becomes small relative to accumulated
  error).

Tests:

(a) per layer ratio (MLP avg over-disp / attn avg over-disp):
```
L     mlp_avg     attn_avg    ratio
 0     1.10         1.66       0.66    (mlp LOWER at L0 — counter-prediction)
 1    11.79         1.69       6.97
 2    11.62         1.72       6.76
 3    10.74         1.70       6.32
 4     3.10         1.42       2.18
 5     1.42         1.57       0.91
18     1.26         1.51       0.83
35     2.20         2.25       0.98
```

(b) MLP-vs-attn gap is biggest at L1-3 — yes, exactly as predicted.

(c) Deep-layer convergence: ratio at L35 = 0.98 (essentially equal).
At L18 ratio = 0.83. **Predicted convergence holds.**

(a) however shows that at L0, MLP over-dispersion is LOWER than
attention's (q baseline is intrinsically high at ~2.2 vs MLP at L0
at ~1.1). So the "MLP > attn" prediction doesn't hold at L0. This
is consistent with: attn_q has its own elevated baseline driven by
something other than within-block ordering; the within-block-ordering
contribution is additive on top, and only becomes visible above
attn's baseline at L1+ where compounded error has built up.

## Counter-evidence considered

1. **Could the L1-3 MLP spike just be from Qwen3-8B's L1-3 weight
   distribution being unusual?** Possible but doesn't explain why
   only MLP and not attention would respond. Both are linear
   projections of the same residual stream (until non-linearities
   apply). The MLP-only response argues for an algorithmic
   asymmetry (ordering or non-linearity sensitivity), not a
   teacher-weight asymmetry.

2. **Could it be SwiGLU's non-linearity amplifying small input
   shifts more than softmax does?** Plausible. SwiGLU has
   gate*silu(x) which is more sensitive to small input shifts than
   attention's softmax+matmul. This could produce MLP-specific
   spikes regardless of within-block ordering. **The bytes don't
   discriminate this from the ordering hypothesis.**

3. **Could it be that the Hassibi-style ℓ∞ optimisation has
   different convergence properties at SwiGLU layers?** Possible.
   The published theory doesn't address layer-type-specific
   behaviour.

## Recipe constraint refinement

Based on this synthesis, a reproduction recipe should:
- Quantise within each block in **attn → MLP order** (not parallel).
- Use a per-block decision step that minimises activation-output
  reconstruction (OBC-style or equivalent), so that compounded
  error from previous tensors feeds into per-block decisions.
- Run sequentially across layers (not all-at-once); within-layer
  ordering matters per (a)-(c) above.

A reproduction's signature SHOULD reproduce:
- L1-3 MLP-only spike (magnitude depends on architecture depth)
- attn-vs-MLP convergence at deep layers
- Approximately full-rank delta everywhere (per-element decisions)
- Modest rank-concentration uptick at L1-2 MLP (10-12% rank-128 %)

A reproduction that quantises tensors in parallel within each block
should produce a SYMMETRIC L1-3 spike at both attn and MLP, which
is NOT what Bonsai shows.

## Discriminator NOT addressed by these bytes

Whether the per-block decisions are made by:
- OBC-style sequential Hessian-aware optimisation
- A LoRA preprocess + per-element SGD-α
- A loss with block-coherence regularisation
- Or some hybrid

The within-block ordering hypothesis is consistent with all of
these. It only constrains the *order* in which decisions are made,
not the per-decision objective.

## Caveats

- The "ratio" table averages over L1-3 (high-spike) layers and
  separates from baseline. The averaging hides per-layer detail.
- Within-block ordering being attn-first is ONE consistent
  interpretation; the bytes don't strictly require it. A
  reproduction-level test (build a recipe with explicit
  attn-before-MLP ordering and check the output's signature) would
  be the direct confirmation.
- All 8B numbers; 1.7B confirmation is qualitative (modest L1 MLP
  elevation) but the L1-3 spike at 1.7B is not large enough to
  produce the same per-layer ratios cleanly. 4B confirmation
  pending (model not on disk in this environment).
